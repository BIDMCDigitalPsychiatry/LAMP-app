Ti.Healthkit.Query

Description

The Ti.Healthkit.Query class is the basis for all the query objects that retrieve data from the HealthKit store. The HKQuery class is an abstract class. You should never instantiate an Ti.Healthkit.Query object directly. Instead, you always work with one of its concrete subclasses:

  • Ti.Healthkit.ObjectQuery
  • Ti.Healthkit.CorrelationQuery
  • Ti.Healthkit.ObserverQuery
  • Ti.Healthkit.SampleQuery
  • Ti.Healthkit.SourceQuery
  • Ti.Healthkit.StatisticsQuery
  • Ti.Healthkit.StatisticsCollectionQuery

Properties

filter : [Ti.Healthkit.Filter][]

A [Ti.Healthkit.Filter][] object used to filter the objects returned from the HealthKit store. (read-only)

If this property is undefined, the query does not filter its results. Instead, it returns all the objects matching the query's other parameters.

sampleType : String

The type of the objects being queried.
