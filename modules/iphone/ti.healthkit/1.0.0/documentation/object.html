Ti.Healthkit.Object

Description

The HealthKit object is a piece of data that can be stored inside the HealthKit store.

This is an abstract class; you always work with one of its concrete subclasses:

  • Ti.Healthkit.CategorySample
  • Ti.Healthkit.Correlation
  • Ti.Healthkit.QuantitySample
  • Ti.Healthkit.Workout

Properties

UUID : String

The universally unique identifier (UUID) for this HealthKit object. (read-only)

HealthKit assigns a UUID to the object when you create it. If you want to add your own unique ID, add it to the object's metadata using the METADATA_KEY_EXTERNAL_UUID key.

metadata : Object

A dictionary containing the metadata for this HealthKit object. (read-only)

The metadata dictionary contains extra information describing this object. The values can be String, Number, or Date objects. For a complete list of predefined metadata keys, see the METADATA_KEY_* constants.

Using predefined keys helps facilitate sharing data between apps; however, you are also encouraged to create your own, custom keys as needed to extend the HealthKit object's capabilities.

You set an object's metadata when you create the object by calling one of these methods (or a related method):

  • Ti.Healthkit.createQuantitySample
  • Ti.Healthkit.createCategorySample
  • Ti.Healthkit.createCorrelation
  • Ti.Healthkit.createWorkout

source: Ti.Healthkit.Source

A HealthKit source, representing the app or device that created this object. (read-only)

The source property is only available on objects you have retrieved from the HealthKit store. When you create a new object, the source is set to undefined. The system automatically sets the source property when you save the object to the HealthKit store. The source is then available the next time the object is retrieved from the store.
