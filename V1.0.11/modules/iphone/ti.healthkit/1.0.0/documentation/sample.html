Ti.Healthkit.Sample

Description

The Ti.Healthkit.Sample class is a subclass of Ti.Healthkit.Object.

A Ti.Healthkit.Sample object represents a piece of data associated with a start and end time. If the sample represents data over a duration, the start time must be earlier than the end time. If the sample represents data at a particular instant, the start and end times can be the same.

This is an abstract class; you always work with one of its concrete subclasses:

  • Ti.Healthkit.CategorySample
  • Ti.Healthkit.Correlation
  • Ti.Healthkit.QuantitySample
  • Ti.Healthkit.Workout

Properties

startDate : Date

The sample's start date. (read-only)

The sample's start date must be equal to or earlier than its end date.

Some samples - for example, body temperature - represent a single point in time. For these samples, both the start and the end date are the same, because they both refer to the point in time when the sample was taken.

Other samples - for example, step count - represent data over a time interval. Here, the sample should use different start and end dates. These dates mark the beginning and end of the sample's time interval, respectively.

endDate : Date

The sample's end date. (read-only)

The sample's end date must be equal to or later than its start date.

Some samples - for example, body temperature - represent a single point in time. For these samples, both the start and the end date are the same, because they both refer to the point in time when the sample was taken.

Other samples - for example, step count - represent data over a time interval. Here, the sample should use different start and end dates. These dates mark the beginning and end of the sample's time interval, respectively.

sampleType : String

The sample type. (read-only)
