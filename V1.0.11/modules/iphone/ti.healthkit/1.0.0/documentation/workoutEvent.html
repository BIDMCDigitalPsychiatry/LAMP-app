Ti.Healthkit.WorkoutEvent

Description

You can use workout events to toggle a workout object between an active and an inactive state. Workouts start in an active state. A pause event switches it to an inactive state; a resume event switches it back to an active state. Adding a pause event when the workout is already inactive, or a resume event when the workout is already active, does not affect the workout's state. These events are ignored.

Properties

date : Date

The time when the transition occurred. (read-only)

For a pause event, this date indicates the start of the break. For a resume event, this date indicates the end of the break. You must use a date between the starting and ending dates of the workout that you intend to modify.

type : Number

The type of workout event. See the WORKOUT_EVENT_TYPE_* constants for possible values.
